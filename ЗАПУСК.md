# OZI BOT — АВТОЗАПУСК ЗА 15 МИНУТ (0 тенге, работает 24/7)

Ничего настраивать не нужно — конфигурация деплоя уже внутри (render.yaml).
Ваши действия: получить токен → загрузить папку → нажать 2 кнопки.

---

## Шаг 1. Токен бота (2 мин, телефон)
1. Telegram → @BotFather → /newbot
2. Имя: **OZI Astana** | Username: **OZI_Astana_bot** (если занято — OZI_KZ_bot)
3. Скопируйте ТОКЕН (вида 7456...:AAF...)
4. Там же: @userinfobot → /start → скопируйте ваш **ID** (цифры)

## Шаг 2. Код на GitHub (5 мин, компьютер)
1. github.com → Sign up (через Google) → кнопка **New repository**
2. Название: `ozi-bot` → **Public** → Create repository
3. Ссылка **«uploading an existing file»** → перетащите ВСЕ файлы из этой папки → **Commit changes**

## Шаг 3. Запуск на Render (5 мин)
1. render.com → **Sign up через GitHub** (даст доступ к репозиторию)
2. **New → Blueprint** → выберите репозиторий `ozi-bot`
3. Render сам прочитает render.yaml и попросит два значения:
   - **BOT_TOKEN** = токен из Шага 1
   - **ADMIN_CHAT_ID** = ваш ID из Шага 1
4. **Apply / Deploy**. Через 2–3 минуты в логах появится «OZI Bot запущен!»
5. Скопируйте адрес сервиса (вида `https://ozi-bot-xxxx.onrender.com`)

## Шаг 4. Чтобы бот не засыпал (3 мин, один раз)
Бесплатный Render усыпляет сервис при простое. Лечится бесплатным пингером:
1. uptimerobot.com → регистрация → **Add New Monitor**
2. Type: HTTP(s) | URL: адрес из Шага 3 | Interval: **5 minutes**
3. Готово — бот работает круглосуточно бесплатно.

## Проверка
Telegram → найдите @ваш_бот → /start → пройдите поиск.
Уведомление о запросе придёт вам в личку (на ADMIN_CHAT_ID).

## Обновление базы центров (потом)
Редактируете `centers_data.py` прямо на GitHub (карандаш → Commit) —
Render сам передеплоит бота за 2 минуты. Ничего перезапускать не надо.

## Если что-то не работает
- Логи: Render → ваш сервис → вкладка **Logs**
- Ошибка «Установите токен» → проверьте BOT_TOKEN в Render → Environment
- Бот молчит после паузы → проверьте, что UptimeRobot-монитор активен
